# AI Worker Language Policy (Changes)

この文書は、Changes リポジトリにおける文書言語ポリシーを AI 作業者向けに定義します。

## 基本方針

- ユーザー向け文書は英語を維持する。
- 開発者向け文書は日本語を優先する。
- AI 作業者向け文書は原則日本語にする。

## 英語を維持すべき要素

次は日本語化せず、原文の英語表記を維持する。

- product 内の機能名・概念名として定義済みの語
- code / schema / CLI / file path / test / artifact / 実機表記と一致が必要な語
- 外部システム・外部仕様・ハードウェアの正式表記
- `validation status`、`analysis status`、`schema value` など機械比較される値
- 翻訳すると一般名詞や別概念と混同しやすい語
- 翻訳で実装・仕様・テスト・UI・ハードウェア表記との対応が不明瞭になる語

## 実務ルール

- 見出し・説明文は日本語化してよい。
- コマンド、オプション名、識別子、定数、型名、API 名は英語のままにする。
- テストが特定文言を比較している場合、当該文言は保持する。
- 迷った場合は「翻訳より整合性」を優先する。

## 変更時チェック

- 変更後に文書関連テストがある場合は実行する。
- 文書言語変更が CLI 実行例や safety 文言を壊していないことを確認する。
- ユーザー向け文書に日本語を混ぜていないことを確認する。
