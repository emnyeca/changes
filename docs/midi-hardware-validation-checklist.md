# MIDI hardware validation checklist

This checklist is for manual validation of real SysEx send to Digitone II.

## Preconditions

- Digitone II is connected through USB MIDI or a MIDI interface.
- The intended Digitone II MIDI output port is visible.
- A known-good `.syx` file has been generated by the product export path.
- Important Digitone II data has been backed up.
- The risk of overwriting pattern data is explicitly accepted.
- Volume and monitoring are in a safe state.
- The validation does not depend on live performance device state.

## Software Preparation

- Install optional MIDI dependencies:
  - `pip install .[midi]`
  - or `pip install mido python-rtmidi`
- Confirm versions:
  - `python -c "import importlib.metadata as md; print('mido', md.version('mido'))"`
  - `python -c "import importlib.metadata as md; print('python-rtmidi', md.version('python-rtmidi'))"`
- Confirm port listing works.
- Confirm dry-run send works.
- Confirm the `.syx` file starts with `0xF0` and ends with `0xF7`:
  - `changes check digitone-syx --syx <file.syx>`

## Manual Validation Steps

1. Generate a `.syx` file through product export.
2. Run `changes check digitone-syx --syx <file.syx>`.
3. Run dry-run send.
4. List available MIDI output ports.
5. Select the Digitone II port explicitly.
6. Confirm the target pattern slot can be overwritten.
7. Run guarded real-send.
8. Confirm import / transfer on Digitone II.
9. Record the result in a dated hardware validation note.
